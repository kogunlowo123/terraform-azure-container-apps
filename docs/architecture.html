Azure Container Apps Environment & Apps Module

Azure Container Apps

Executive Summary

Production-ready Terraform module for deploying Azure Container Apps with fully managed environments, multiple container apps, Dapr sidecar components, custom domains with managed certificates, workload profiles, VNet injection, internal/external ingress with traffic splitting, IP security restrictions, health probes (liveness/readiness), volume mounts from Azure File Shares, container registries, secrets management, and integrated Log Analytics monitoring. Supports zero-to-N scale-to-zero serverless workloads.

Data Flow

1
Client requests arrive at the environment's ingress controller (external or internal LB).
2
Traffic weights split requests across revisions for canary/blue-green deployments.
3
Container apps scale from 0 to max replicas based on HTTP traffic or custom rules.
4
Dapr sidecars provide state management, pub/sub messaging, and service invocation.
5
Volume mounts provide persistent storage from Azure File Shares.
6
All application logs stream to Log Analytics workspace.

Production Readiness Checklist

  • Enable VNet injection with dedicated /21 subnet
  • Enable zone redundancy for high availability
  • Configure workload profiles for dedicated compute
  • Set up health probes (liveness and readiness)
  • Configure IP security restrictions on ingress
  • Set min_replicas > 0 for critical services
  • Use managed identity for registry authentication
  • Configure custom domains with managed certificates
  • Set appropriate Log Analytics retention
  • Use traffic splitting for safe deployments
